VULNERABILITY DETAILS



<b class="line"><!---->VULNERABILITY DETAILS<!----></b>



<br>





In this report, we described how the PLRU-based cache side channel introduced in the leaky.page PoC can be improved and extended for cross-core and cross-process attacks.



<span class="line"><!---->In this report, we described how the PLRU-based cache side channel introduced in the leaky.page PoC can be improved and extended for cross-core and cross-process attacks.<!----></span>



<br>





The attached PoC code generates an eviction set for the last-level cache (LLC) using only primitives available in a restricted Chrome setting (degraded performance.now, no shared memory).



<span class="line"><!---->The attached PoC code generates an eviction set for the last-level cache (LLC) using only primitives available in a restricted Chrome setting (degraded performance.now, no shared memory).<!----></span>



<br>





This eviction set can further be used to carry out cross-core side-channel attacks or can be used as a building block for Spectre and Rowhammer attacks, both against isolated websites and native processes.



<span class="line"><!---->This eviction set can further be used to carry out cross-core side-channel attacks or can be used as a building block for Spectre and Rowhammer attacks, both against isolated websites and native processes.<!----></span>



<br>



<br>





We plan on releasing the findings in this report as part of an academic paper. You can find a current draft of our paper attached to this report with additional details of our research (the relevant parts are sections 4.1, 5.2 and 6.3).



<span class="line"><!---->We plan on releasing the findings in this report as part of an academic paper. You can find a current draft of our paper attached to this report with additional details of our research (the relevant parts are sections 4.1, 5.2 and 6.3).<!----></span>



<br>



<br>





VERSION



<b class="line"><!---->VERSION<!----></b>





 (most likely applicable to all Chrome versions and Intel platforms, with some modifications to the PoC)



<span class="line"><!----> (most likely applicable to all Chrome versions and Intel platforms, with some modifications to the PoC)<!----></span>



<br>





Chrome Version: 108.0.5359.94 stable



<span class="line"><!---->Chrome Version: 108.0.5359.94 stable<!----></span>



<br>





Operating System: Ubuntu 22.04.1 LTS (Linux 5.15.0-56)



<span class="line"><!---->Operating System: Ubuntu 22.04.1 LTS (Linux 5.15.0-56)<!----></span>



<br>





CPU: Intel Core i5-8250U (we have also had successful runs with the same PoC on an i7-8665U)



<span class="line"><!---->CPU: Intel Core i5-8250U (we have also had successful runs with the same PoC on an i7-8665U)<!----></span>



<br>



<br>





REPRODUCTION CASE



<b class="line"><!---->REPRODUCTION CASE<!----></b>



<br>





Our PoC presents a number of improvements over the PLRU-based leakage of leaky.page [0].



<span class="line"><!---->Our PoC presents a number of improvements over the PLRU-based leakage of leaky.page [0].<!----></span>



<br>





This allows an attacker to amplify a single cache event (hit/miss) to make it detectable using the restricted timing primitives in Chrome.



<span class="line"><!---->This allows an attacker to amplify a single cache event (hit/miss) to make it detectable using the restricted timing primitives in Chrome.<!----></span>



<br>



<br>





In leaky.page, an attacker can detect a single eviction from the L1d cache using a low-precision timer.



<span class="line"><!---->In leaky.page, an attacker can detect a single eviction from the L1d cache using a low-precision timer.<!----></span>



<br>





We can, additionally, also detect:



<span class="line"><!---->We can, additionally, also detect:<!----></span>



<br>



<br>





    - the order of accesses to lines in the L1d cache



<span class="line"><!---->    - the order of accesses to lines in the L1d cache<!----></span>



<br>





    - evictions from the last-level cache (LLC), which is shared across cores



<span class="line"><!---->    - evictions from the last-level cache (LLC), which is shared across cores<!----></span>



<br>



<br>





The former capability (1.) uses the L1d replacement policy, but prepares the cache state such that the time of the traversal is conditioned on the order of two accesses. Please find the technical details in section 4.1 of the attached paper.



<span class="line"><!---->The former capability (1.) uses the L1d replacement policy, but prepares the cache state such that the time of the traversal is conditioned on the order of two accesses. Please find the technical details in section 4.1 of the attached paper.<!----></span>



<br>





The latter capability (2.) is achieved by converting the LLC eviction event into an L1d reordering event. Because of out-of-order execution, instructions may be reordered depending on how long it takes for their dependencies to resolve. Therefore, we can condition the execution order of two L1 loads (which we can detect with 1.) on whether another cache line hits or misses in the LLC.



<span class="line"><!---->The latter capability (2.) is achieved by converting the LLC eviction event into an L1d reordering event. Because of out-of-order execution, instructions may be reordered depending on how long it takes for their dependencies to resolve. Therefore, we can condition the execution order of two L1 loads (which we can detect with 1.) on whether another cache line hits or misses in the LLC.<!----></span>



<br>



<br>





Orthogonally, we have made some robustness improvements to the original traversal patterns.



<span class="line"><!---->Orthogonally, we have made some robustness improvements to the original traversal patterns.<!----></span>



<br>



<br>





Our PoC demonstrates the creation of LLC eviction sets using this primitive. LLC eviction sets are the fundamental building block for cross-core side-channel attacks.



<span class="line"><!---->Our PoC demonstrates the creation of LLC eviction sets using this primitive. LLC eviction sets are the fundamental building block for cross-core side-channel attacks.<!----></span>



<br>





The algorithm used for finding the eviction set is based on Vila et al. [1].



<span class="line"><!---->The algorithm used for finding the eviction set is based on Vila et al. [1].<!----></span>



<br>





The heart of the exploit, i.e., the traversal pattern for timing cache accesses, can be found in amplify.js, and instructions for running the PoC can be found in the readme file.



<span class="line"><!---->The heart of the exploit, i.e., the traversal pattern for timing cache accesses, can be found in amplify.js, and instructions for running the PoC can be found in the readme file.<!----></span>



<br>



<br>





To run the PoC, the associativity form field must first be set to the CPU’s LLC associativity (usually 16 or 12). During the run, the console will show the progress of finding an eviction set.



<span class="line"><!---->To run the PoC, the associativity form field must first be set to the CPU’s LLC associativity (usually 16 or 12). During the run, the console will show the progress of finding an eviction set.<!----></span>



<br>





On our machine, the LLC eviction set is found in ~25 seconds with a success rate of ~70%, but we have not exhausted all optizimation opportunities. The PoC may require some platform-dependent parameter tweaking: we have not yet expended efforts towards making the PoC portable. Consider tweaking the initial block count if no eviction sets are found with the initial configuration.



<span class="line"><!---->On our machine, the LLC eviction set is found in ~25 seconds with a success rate of ~70%, but we have not exhausted all optizimation opportunities. The PoC may require some platform-dependent parameter tweaking: we have not yet expended efforts towards making the PoC portable. Consider tweaking the initial block count if no eviction sets are found with the initial configuration.<!----></span>



<br>



<br>





After obtaining the ability to construct LLC eviction sets, the attacker has the following capabilities:



<span class="line"><!---->After obtaining the ability to construct LLC eviction sets, the attacker has the following capabilities:<!----></span>



<br>



<br>





    - perform LLC-based side-channel attacks (i.e., cross-core and cross-process)



<span class="line"><!---->    - perform LLC-based side-channel attacks (i.e., cross-core and cross-process)<!----></span>



<br>





    - increase the leaky.page amplification by using LLC-congruent addresses, for which L1 misses receive a larger miss penalty, increasing the amplification factor



<span class="line"><!---->    - increase the leaky.page amplification by using LLC-congruent addresses, for which L1 misses receive a larger miss penalty, increasing the amplification factor<!----></span>
