VULNERABILITY DETAILS



<b class="line"><!---->VULNERABILITY DETAILS<!----></b>



<br>





EventSource cross-origin requests are not blocked by CORB, even though the response Content-Type is set to text/html. This means the renderer could access cross-origin response headers.



<span class="line"><!---->EventSource cross-origin requests are not blocked by CORB, even though the response Content-Type is set to text/html. This means the renderer could access cross-origin response headers.<!----></span>



<br>



<br>





This vulnerability is similar to the one reported in WebSockets in 2019 in

https://bugs.chromium.org/p/chromium/issues/detail?id=944619

 which is now fixed.
