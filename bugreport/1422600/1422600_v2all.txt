VULNERABILITY DETAILS



<b class="line"><!---->VULNERABILITY DETAILS<!----></b>



<br>



<br>





*msm_ringbuffer_new* is responsible for creating ringbuffer. It will call *msm_fence_context_alloc* to create context, and directly assigned[1] the return value to *ring->fctx* without any check. But in fact *msm_fence_context_alloc* may fail and return ERR_PTR[2]. There will be problems when accessing *ring->fctx*[3] in *msm_ringbuffer_destroy*.



<span class="line"><!---->*msm_ringbuffer_new* is responsible for creating ringbuffer. It will call *msm_fence_context_alloc* to create context, and directly assigned[1] the return value to *ring-&gt;fctx* without any check.
